Guía de diseño · Para el equipo de desarrollo

La identidad de Feria Mendoza, traducida a valores exactos.

Este documento es la fuente de verdad visual del eCommerce. Define colores, tipografía, espaciado y componentes con valores listos para código, y marca las reglas de accesibilidad que deben cumplirse. La idea: que cualquier pantalla nueva se sienta parte de Feria Mendoza sin tener que adivinar.

🎨 Paleta con contraste verificado (WCAG 2.1) 🧩 8 componentes especificados ♿ Objetivo: nivel AA 📐 Base de espaciado: 4px
01

Color

La paleta nace del anillo multicolor y el logo de la marca. Cada color tiene un rol asignado y un ratio de contraste verificado. Regla de oro: el color no comunica solo por sí mismo (un daltónico no distingue "rojo = error"), por eso siempre se acompaña de texto o ícono.

Elevación

Sombras cálidas y suaves (tintadas con el color tinta, no negro puro) para dar profundidad sin ensuciar. Tres niveles, nada más.

05

Iconografía

Íconos de trazo (line), no rellenos, para dialogar con el logo lineal de la percha. Recomiendo una sola librería para todo el sitio y no mezclar estilos.

  • Librería sugeridaLucide o Phosphor (weight "regular"). Open source y consistentes.
  • Grilla24×24px de diseño. Tamaños en uso: 20 / 24 / 28px.
  • Grosor de trazo1.5–1.75px, uniforme en todo el set.
  • ColorHeredan currentColor del texto que acompañan.
  • Área táctilSi el ícono es un botón, su zona clickeable es 44×44px aunque el dibujo mida 24px.
  • ReglaUn ícono nunca va solo como única etiqueta de una acción importante; se acompaña de texto o aria-label.

09

Formularios

Etiqueta siempre visible arriba del campo (nunca solo placeholder). El error explica qué pasó y cómo resolverlo, con texto + color + ícono.

Como querés que aparezca en tu publicación.
Falta el @ en el correo. Ej: nombre@mail.com
  • Alto de campo≥44px · padding 12px 14px · radio 12px.
  • Focoborde --azul-deep + halo 0 0 0 3px rgba(62,82,144,.18).
  • Labelsiempre presente y asociada con for / id.
  • Errorborde carmín + mensaje con aria-describedby y aria-invalid="true".
  • Tipos correctostype="email", tel, inputmode → mejor teclado en móvil.

12

Accesibilidad · checklist AA

Requisitos mínimos no negociables. Cumplirlos amplía el público comprador (incluye gente mayor y con baja visión, muy presente en una feria familiar) y además mejora el SEO.

  • 1
    Contraste de texto ≥ 4.5:13:1 para texto grande (≥24px) y para bordes de controles. La paleta de esta guía ya lo cumple.
  • 2
    Foco siempre visibleTodo elemento interactivo muestra un anillo de foco (azul profundo, offset 2px) al navegar con teclado. Prohibido outline:none sin reemplazo.
  • 3
    Área táctil ≥ 44×44pxBotones, íconos-acción y links del menú. Evita toques errados en el celular.
  • 4
    HTML semántico<button> para acciones, <a> para navegar, <nav>, <main>, encabezados en orden (h1→h2→h3).
  • 5
    Imágenes con altCada foto de prenda describe qué es. Las decorativas llevan alt="".
  • 6
    Formularios etiquetados<label for> en cada campo; errores con aria-describedby y aria-invalid.
  • 7
    El color no es el único mensajeEstados y badges combinan color + ícono o texto (clave para daltonismo).
  • 8
    Navegable con tecladoTodo el flujo de compra se completa con Tab/Enter. Orden de foco lógico. Modales atrapan el foco y cierran con Esc.
  • 9
    Respeta prefers-reduced-motionAnimaciones suaves y desactivables para quien lo configura.
  • 10
    Zoom hasta 200%El layout no se rompe; sin scroll horizontal. Texto base 16px.
🧪
Herramientas de verificación para el equipo: axe DevTools, Lighthouse (pestaña Accessibility) y navegación real solo con teclado antes de cada release.

Entrega · tokens listos para código

Copiá este bloque como punto de partida. Define todos los tokens como variables CSS; el resto del sitio los referencia. Si usan Tailwind, se mapean 1:1 en theme.extend.

/* tokens.css — Feria Mendoza */ :root{ /* Marca */ --terracota: #BC5B2E; --terracota-deep: #A84A22; --azul: #4E63A8; --azul-deep: #3E5290; --menta: #4FA98C; --menta-deep: #2E7D63; --ciruela: #6E4E96; --carmin: #B23A3A; --lavanda: #C4C4DD; /* Neutros + superficies */ --ink: #1F1B24; --ink-soft: #4A4550; --border: #E4E0E8; --surface: #FFFFFF; --bg: #FBF9F5; --bg-subtle: #F4F1F6; /* Estado */ --success: #2E7D63; --error: #B23A3A; --info: #4E63A8; /* Radios */ --r-sm: 8px; --r-md: 12px; --r-lg: 16px; --r-xl: 24px; --r-pill: 999px; /* Espaciado (base 4px) */ --s-1:4px; --s-2:8px; --s-3:12px; --s-4:16px; --s-6:24px; --s-8:32px; /* Sombras */ --sh-sm: 0 1px 2px rgba(31,27,36,.06); --sh-md: 0 4px 14px rgba(31,27,36,.08); --sh-lg: 0 14px 36px rgba(31,27,36,.12); /* Tipografía */ --font-display: 'Fredoka', sans-serif; --font-body: 'Inter', sans-serif; --font-mono: 'JetBrains Mono', monospace; }
📦
Fuentes vía Google Fonts: Fredoka (400–700), Inter (400–700), JetBrains Mono (400–600). Autohospedar las fuentes mejora la velocidad de carga y la privacidad.
